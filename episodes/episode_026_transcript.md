[NOVA]: There is a new paper on arXiv called MegaTrain, and I want to just state the core claim before we unpack it. MegaTrain can train models with over 100 billion parameters on a single GPU. One GPU. Not a cluster. Not a specialized machine. One GPU.

[ALLOY]: I want to make sure people understand why that sounds impossible and then understand why it is actually working. The traditional constraint on training large models has been VRAM. GPU memory holds the model weights, the optimizer state, the gradients, and the activations for each training step. At 100 billion parameters, the numbers are staggering. Even with 16-bit precision, you are talking about 200 gigabytes just for the weights. The optimizer state for a 100-billion-parameter model trained with Adam is roughly three times the model size — 600 gigabytes. No current GPU has that much VRAM. The standard solution is model parallelism: shard the model across multiple GPUs, pipeline the computation, and accept the communication overhead.

[NOVA]: MegaTrain takes a different approach. Instead of distributing the model across GPUs, it keeps the model weights and optimizer state in host CPU memory and treats the GPUs as disposable compute units. When a GPU needs to do a piece of computation, it streams the relevant weights from CPU memory, performs the arithmetic, and streams the results back. The GPU is not storing the model — it is just doing the math on data it fetches on demand.

[ALLOY]: And the reason this is becoming viable now is that GPU compute density has been advancing faster than GPU memory capacity. Bandwidth between CPU and GPU has been improving significantly with newer architectures like the GH200, which combines CPU and GPU memory in a unified pool. The H200 has 1.5 terabytes of CPU memory available for the system, and the bandwidth between CPU and GPU on that platform is high enough that streaming weights in and out is not as punishing as it would have been on older hardware.

[NOVA]: They demonstrate training a 120-billion-parameter model on a single H200 with 1.5 terabytes of system RAM. That is a lot of RAM and it is not cheap, but a single H200 is a commercially available machine. You do not need a data center. You do not need a cluster. You need one very expensive server. And that is a meaningful change in the accessibility of frontier training.

[ALLOY]: For smaller models, the numbers are actually impressive in terms of throughput. MegaTrain achieves 1.84 times faster training than DeepSpeed ZeRO-3 on a 14-billion-parameter model. ZeRO-3 is one of the most widely used approaches for distributed training of large models. Beating it by nearly 2x on a single machine on a smaller scale model suggests the streaming architecture has genuine efficiency advantages, not just memory advantages.

[NOVA]: But the number that I keep thinking about is the 7-billion-parameter model trained with 512,000 tokens of context on a single GH200. 512,000 tokens. That is a frontier-scale context window. Gemini 1.5 Pro launched with a 1-million-token context window and it was considered a landmark achievement. MegaTrain is demonstrating that you can train a model with half a million tokens of context on one chip. One GH200. No cluster. No InfiniBand networking. No special interconnect. One chip.

[ALLOY]: The implication is a significant democratization of frontier research capability. A university lab, an independent researcher, a small company — they can now run experiments that previously required a cluster of H100s. The cost curve for this kind of training is changing. It is not free, and it is not trivial, but the gap between what you can do on a research budget and what you can do at a major lab is getting meaningfully smaller.

[NOVA]: And it has implications for the concentration of AI capability more broadly.
