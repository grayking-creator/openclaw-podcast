[ALLOY]: Das Agent-to-Agent Protocol erreichte v1.0 im Jahr 2026 unter der Linux Foundation und etablierte eine formale Spezifikation dafür, wie Agents aus verschiedenen Frameworks einander entdecken, Kommunikationskanäle aufbauen und Aufgaben delegieren. Das Protokoll definiert „Agent Cards" — JSON-Capability-Manifeste — für die Agent-Discovery und eine aufgabenbasierte State Machine für langlebige Interaktionen unter Verwendung von JSON-RPC 2.0.

[NOVA]: Ursprünglich von Google gestartet, wird A2A nun von der Linux Foundation gemeinsam mit MCP regiert. Der MCP- versus A2A-Unterschied ist das zentrale mentale Modell. MCP standardisiert, wie ein Agent sich mit externen Tools, Datenbanken und Datenquellen verbindet — es geht darum, was ein Agent tun kann. A2A standardisiert, wie Agents miteinander kommunizieren — es geht darum, wie Agents zusammenarbeiten.

[ALLOY]: MCP ist bereits weit verbreitet im Agent-Stack. A2A v1.0 ist das komplementäre Protokoll, das plattformübergreifende Agent-Übergabe ohne Custom-Integrationscode für jedes Paar ermöglichen wird. Der Unterschied ist wichtig für Builder. Wenn du einen Agent mit einer Datenbank verbindest, nutzt du MCP. Wenn du zwei Agents dazu bringst, miteinander zu sprechen, nutzt du A2A.

[NOVA]: Eine Agent Card ist ein JSON-Manifest, das beschreibt, was ein Agent tun kann. Die Karte enthält die Identität des Agents, seine Capabilities, die Protokolle, die er spricht, die Authentifizierungsmechanismen, die er unterstützt, und die Aufgabentypen, die er akzeptiert. Wenn ein Agent Arbeit an einen anderen delegieren möchte, entdeckt er die Karte des Zielagents und nutzt sie, um herauszufinden, wie kommuniziert werden soll.

[ALLOY]: Die Task State Machine ist die andere Hälfte des Protokolls. Eine langlebige Aufgabe hat einen definierten Lebenszyklus: eingereicht, arbeitend, Input-erforderlich, abgeschlossen, fehlgeschlagen, abgebrochen. Die State Machine definiert, was jeder Status bedeutet, welche Übergänge gültig sind, und welche Daten die Agents bei jedem Übergang austauschen. Die State Machine ist das, was einen Multi-Agent-Workflow robust gegen partielles Versagen macht.
