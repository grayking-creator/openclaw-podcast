[02:00] Inmersión profunda en OpenClaw v2026.6.1 — Workboard, Skill Workshop, recuperación SQLite y expansión de proveedores

OpenClaw v2026.6.1 aterrizó el 3 de junio de 2026 y es el bloque principal del episodio. La incorporación principal es Workboard — una superficie de planificación y seguimiento de ejecuciones multi-agente con primitivas de orquestación, ejecuciones de tableros respaldadas por tareas y comentarios de tareas en el modal de edición. Workboard no es solo una lista de tareas; es una superficie de coordinación completa que puede impulsar la descomposición de tareas y ejecución de árbol de trabajo por tarea con el cableado correcto del agente. El cambio clave es que OpenClaw se mueve de un runtime de agente único hacia una capa de coordinación multi-agente. Un tablero kanban en Workboard no es solo seguimiento de estado — impulsa la descomposición real de tareas con herramientas de coordinación de agentes para planificación y seguimiento de ejecuciones multi-agente.

Skill Workshop añade autoría de habilidades gobernadas a la UI de Control. Las propuestas fluyen a través de controles de aplicar/rechazar/cuarentena con metadatos de reversión. Los archivos de soporte bajo carpetas de habilidades estándar obtienen protecciones de escaneo, hash y reversión. La herramienta de agente skill_workshop ahora es parte del conjunto de herramientas de agente de OpenClaw — un agente puede iniciar una propuesta de habilidad, obtenerla revisada y aprobada todo dentro del runtime de OpenClaw sin herramientas externas. La herramienta de agente skill_workshop puede aplicar, rechazar y poner en cuarentena propuestas explícitas a través del flujo de revisión protegido. Las propuestas pueden llevar archivos de soporte aprobados bajo carpetas de habilidades estándar con protecciones de escaneo, hash y reversión. Las propuestas pendientes pueden ser revisadas en su lugar con frontmatter de propuesta versionado y fechado antes de la aprobación. La guía de Skill Workshop también aparece en los prompts de app-server de Codex cuando skill_workshop está disponible, lo que significa que Codex puede solicitar gobernanza de habilidades en el momento adecuado durante una ejecución. La UI de Control ahora tiene un flujo de Skill Workshop más completo con listas de propuestas, acciones del día, transferencia de revisiones, vistas previas de archivos buscables, estados de revisión, cobertura de locale y enrutamiento de sesión reutilizable.

La recuperación del runtime de agente y CLI es la tercera área de mejora principal. Las llamadas a herramientas que se interrumpen, enlaces de sesión obsoletos, transferencias de compactación y reintentos de entrega de medios se recuperan más limpiamente. Las correcciones cubren llamadas a herramientas interrumpidas, enlaces de sesión obsoletos, transferencias de compactación y reintentos de entrega de medios en cinco pull requests. Las rutas de solicitud de proveedores y plugins vincularon más temporizadores, reintentos,生命周期 de OAuth y código de dispositivo, descargas de medios, sondeos de servicios locales y rutas de polling de contenido generado antes de que puedan bloquear una ejecución.

El estado del monitor de iMessage, las colas entrantes y los ledgers de instalación de plugins se movieron a estado respaldado por SQLite para que los reinicios se recuperen sin escaneo duplicado del sistema de archivos. El índice de instalación de plugins ahora se persiste en SQLite para que la búsqueda de paquetes instalados sobreviva a recargas con menos escaneo del sistema de archivos. iOS añade valores predeterminados de relé de push alojado, reproducción realtime de Talk y una ruta de ping de WebSocket protegida para sesiones móviles más confiables bajo condiciones de red variables. La entrega por canal y móvil a través de Telegram, WhatsApp, iMessage, Slack, Discord, Microsoft Teams, Google Chat, Google Meet y Talk realtime de iOS es más estable con correcciones a las colas entrantes y relé de push.

La expansión de proveedores añade MiniMax M3 al registro de modelos con su ventana de contexto de 1,000,000 de tokens, arquitectura de atención dispersa (MSA), capacidades multimodales, MiniMax Code y disponibilidad completa de API — los agentes gestionados por OpenClaw pueden enrutarse a él a través de la interfaz de proveedor estándar sin configuración personalizada. Los endpoints de OAuth de cuenta ahora se incluyen en los metadatos del proveedor para flujos de OAuth más completos. Las correcciones de catálogo de Google y Vertex abordan la navegación de modelos y la carga de catálogo con reconocimiento de credenciales. OpenRouter ahora usa SQLite para el caché de metadatos de modelos, lo que significa que el estado del catálogo sobrevive a los reinicios y evita llamadas de red repetidas para metadatos de catálogo. Las capacidades de Copilot Claude 1M son reconocidas. La alineación de Foundry reasoning aborda los modelos de razonamiento a través del endpoint de Foundry de Microsoft. Las protecciones de respuesta de replay de OpenAI protegen contra casos extremos de replay no determinístico. Las habilidades, metadatos de sesión, estado del runtime de gateway, metadatos de plugins, watchers de memoria y escrituras de store hacen menos trabajo repetido en rutas frecuentes. El modo de código añade espacios de nombres internos para sesiones de agente con ámbito y globales con dispatch de herramientas de espacio de nombres exacto, más archivos y docs de API de MCP para integraciones de modo de código.

[26:00] Claude Code 2.1.162 — waitingFor JSON, reglas de permisos, correcciones de timeout de MCP y manejo de interrupciones

Claude Code 2.1.162 está cinco releases después de la línea base de EP059 de 2.1.157. El cambio más relevante para operadores es que `claude agents --json` ahora muestra un campo `waitingFor` — te dice por qué una sesión en segundo plano está bloqueada, ya sea un prompt de permiso u otro bloqueador. Esto hace posible construir herramientas de monitoreo que diagnostican bloqueos sin requerir que un humano se conecte e inspeccione.

Las reglas de permisos de WebFetch ahora son explícitas — las reglas de denegar y preguntar tienen precedencia sobre el auto-aprobado de hosts preaprobados, lo que cierra un vacío donde un dominio podía estar en la lista preaprobada y aún tener reglas explícitas que deberían haberlo bloqueado. El manejo de rutas de Windows para barras invertidas y rutas con variante de mayúsculas/minúsculas está corregido — Las reglas de denegar lectura ahora realmente ocultan archivos de los resultados de Glob y Grep. Las configuraciones de timeout de MCP por debajo de 1000ms ya no se redondean a 1 segundo y no abortan cada llamada a herramienta; caen al valor predeterminado o `MCP_TOOL_TIMEOUT`, y `claude mcp get` los anota correspondientemente. La corrección de interrupción al inicio de turno cierra un vacío en sesiones de stream-json y SDK donde un Esc presionado inmediatamente después de enviar un prompt descartaría silenciosamente el turno y lo dejaría ejecutándose sin retroalimentación.

`/effort` ahora confirma cuando el nivel elegido persiste como predeterminado para nuevas sesiones. Remote Control se muestra como una pastilla persistente de pie de página con un enlace de sesión — ya no aparece como un mensaje de inicio que se desplaza. El rebranding de Windsurf a Devin se refleja en `/ide`, `/terminal-setup` y `/scroll-speed`. Un bloqueo silencioso de inicio cuando el directorio de configuración es de solo lectura o no escribible ahora muestra un diagnóstico en lugar de una pantalla en blanco. Los grupos de inicio más silenciosos notifican por severidad y la información de sesión y anuncios comparten una sola línea por lanzamiento.

La operación `workspaceSymbol` de la herramienta LSP ahora acepta un parámetro de consulta y lo pasa al servidor de lenguaje. Los errores 400 de API `no low surrogate in string` para side-queries de clasificadores y descripciones de servidores MCP con emoji cerca de los límites de truncamiento están corregidos. Varios problemas de UI de `claude agents` están resueltos: el detalle de estado ya no trunca en 60–120 columnas en terminales anchos, los nombres largos de sesión ya no truncan en 40 columnas, attach ya no rebota de vuelta a la lista de sesiones en el primer intento después de un reinicio de servicio en segundo plano, Ctrl+V pegar imagen funciona en el input de dispatch y el cuadro de respuesta de sesión, y el envío a segundo plano con ← ya no pierde silenciosamente la conversación cuando el servicio en segundo plano no puede iniciar. Los turnos fallidos ahora muestran una línea de advertencia compacta en lugar de un bloque de error rojo multilínea.

Lista de pruebas prácticas: ejecuta `claude agents --json` en una sesión en segundo plano y verifica `waitingFor`; verifica el comportamiento de denegación explícita de WebFetch; prueba un servidor MCP con un timeout por debajo de 1000ms y confirma que está anotado en `claude mcp get`; envía una interrupción al inicio de un turno en una sesión de stream-json y verifica la retroalimentación de Interrumpido.

[40:00] Qwen 3.7 Max y Plus — división de razonamiento-codificación vs visión-multimodal en la frontera de agentes

Qwen 3.7 aterrizó el 19 de mayo de 2026 como dos variantes distintas. Max apunta a razonamiento, codificación y matemáticas — el mismo carril que Codex y los modelos principales de Claude Code. Plus apunta a tareas multimodales y de visión. Ambos son de peso abierto y disponibles a través de API de OpenRouter y herramientas de despliegue local. El posicionamiento agentic de Qwen significa que los modelos están diseñados para uso de herramientas e interacción agente-computadora, no solo para completar turnos únicos.

Para operadores de OpenClaw, Codex y Claude Code, Qwen 3.7 Max es una opción de enrutamiento para cargas de trabajo intensivas en código. Para agentes que necesitan manejar imágenes, documentos y entradas visuales, Qwen 3.7 Plus llena un carril diferente que los modelos puros de codificación. La división importa para las decisiones de enrutamiento de modelos — eliges basándote en si la tarea es principalmente generación de código o comprensión multimodal.

Pruébalo: descarga una cuantización de Qwen 3.7 a través de Ollama o LM Studio y compara la calidad de generación de código contra tu modelo actual de agente de codificación. Para tareas multimodales, prueba Qwen 3.7 Plus a través de API de OpenRouter con una entrada de documento o imagen.

[51:00] agentmemory — contexto persistente compartido para toda la pila de agentes

agentmemory resuelve el problema de la pizarra en blanco en cada agente que ejecutes. Cuando Claude Code, Codex, Hermes y OpenClaw comparten el mismo servidor de memoria en el puerto 3111, el contexto del trabajo de la semana pasada sobrevive en la sesión de esta semana. El sistema utiliza el motor iii, se instala como un paquete global de npm y se conecta mediante plugins nativos, hooks o MCP dependiendo del agente.

El caso de uso más concreto es un desarrollador que trabaja en una base de código en múltiples agentes — Claude Code para refactorización de alto nivel, Codex para operaciones de shell y archivos, Hermes para orquestar tareas de múltiples pasos. Sin una capa de memoria compartida, cada agente comienza frío. Con agentmemory, el contexto de la última sesión de Claude Code informa lo que Codex hace a continuación.

Instala, ejecuta el demo, conéctalo a tu agente y verifica la recuperación de memoria entre dos sesiones.

[61:00] CodeGraphContext v0.4.13 — grafos de código para agentes y desarrolladores

CodeGraphContext indexa código en una base de datos de grafos y expone el grafo tanto a través de una CLI como de un servidor MCP. El diseño de modo dual significa que un solo índice sirve tanto para la exploración humana como para las llamadas de herramientas de agentes. El motor de indexación analiza el código usando nodos tree-sitter y construye un grafo de componentes — funciones, clases, métodos, parámetros, relaciones de herencia, llamadas de funciones e imports. El modo CLI permite a los desarrolladores ejecutar consultas en lenguaje natural contra el grafo. El modo MCP expone herramientas como `get_callers`, `get_callees`, `get_class_hierarchy` y `get_call_chain` a través del protocolo MCP. La vigilancia de archivos en vivo actualiza el grafo en tiempo real a medida que los archivos cambian. Los bundles preindexados te permiten cargar repositorios famosos instantáneamente sin necesidad de indexar. El grafo reduce la sobrecarga de llamadas de herramientas en bases de código más grandes donde una búsqueda de texto tendría que ejecutarse en docenas de archivos. Conéctalo a tu agente y haz la misma pregunta a través de la interfaz MCP para comparar la respuesta del agente contra lo que obtuviste desde la CLI.
