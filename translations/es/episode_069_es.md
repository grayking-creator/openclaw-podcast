[ALLOY]: Un nuevo participante en el espacio de frameworks de agentes ha llamado la atención de la comunidad: Apache Burr. El enlace completo está en las notas del programa, y se posiciona como un framework enfocado en la confiabilidad. Mientras muchos frameworks se centran en qué tan rápido puedes construir un demo, Burr se enfoca en los puntos débiles de producción: flujos de trabajo de larga duración, ejecución con estado y recuperación de fallos.

[NOVA]: La filosofía central de Burr es que un agente es esencialmente una máquina de estados. Separa la "lógica de decisión" de la "infraestructura con estado". Cada paso en un flujo de trabajo agéntico —desde el prompt inicial hasta una llamada a herramienta hasta la salida final— es una acción nombrada con su propio estado. Este estado se captura en cada paso y se almacena en un backend de persistencia conectable como Postgres o SQLite.

[ALLOY]: Este modelo de "ejecución duradera" cambia las matemáticas para los reintentos. En un bucle de agente típico, si una llamada a herramienta falla o el modelo devuelve JSON malformado, quizás tengas que reiniciar toda la secuencia desde el principio. Eso es costoso y lento. Con Burr, puedes reanudar desde el último checkpoint exitoso. Si la cuarta llamada a herramienta en una secuencia de diez falla, solo tienes que reintentar ese cuarto paso. Esta es una victoria masiva para la confiabilidad, especialmente cuando estás llamando APIs de pago que cobran por token.

[NOVA]: El framework también incluye una UI de observabilidad integrada que te permite visualizar toda la traza de decisiones. Puedes ver qué transiciones se tomaron, cuál fue la salida del modelo en cada etapa y dónde se mutó el estado. Esto hace que la depuración de agentes complejos de múltiples pasos sea mucho más manejable. También es consciente de async y está diseñado para ejecutarse en modo servidor, permitiendo que múltiples clientes se coordinen alrededor de una ejecución de agente única.

[ALLOY]: El hilo que importa para los constructores no es abstracto. Se trata de dónde se sitúa la autoridad.
