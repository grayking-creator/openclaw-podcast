[ALLOY]: El Protocolo de Agente a Agente alcanzó la v1.0 en 2026 bajo la Linux Foundation, estableciendo una especificación formal para cómo los agentes de diferentes frameworks se descubren entre sí, establecen canales de comunicación y delegan tareas. El protocolo define "tarjetas de agente" — manifiestos de capacidades en JSON — para el descubrimiento de agentes, y una máquina de estados basada en tareas para interacciones de larga duración usando JSON-RPC 2.0.

[NOVA]: Originalmente lanzado por Google, A2A ahora está gobernado por la Linux Foundation junto con MCP. La distinción MCP versus A2A es el modelo mental clave. MCP estandariza cómo un agente se conecta a herramientas externas, bases de datos y fuentes de datos — se trata de lo que un agente puede hacer. A2A estandariza cómo los agentes se comunican entre sí — se trata de cómo los agentes trabajan juntos.

[ALLOY]: MCP ya está ampliamente adoptado en el stack de agentes. A2A v1.0 es el protocolo complementario que permitirá el traspaso de agentes entre frameworks sin código de integración personalizado para cada par. La distinción importa para los constructores. Si estás conectando un agente a una base de datos, estás usando MCP. Si estás conectando dos agentes para que hablen entre sí, estás usando A2A.

[NOVA]: Una tarjeta de agente es un manifiesto JSON que describe lo que un agente puede hacer. La tarjeta incluye la identidad del agente, sus capacidades, los protocolos que habla, los mecanismos de autenticación que soporta y los tipos de tareas que acepta. Cuando un agente quiere delegar trabajo a otro, descubre la tarjeta del agente objetivo y la usa para determinar cómo comunicarse.

[ALLOY]: La máquina de estados de tareas es la otra mitad del protocolo. Una tarea de larga duración tiene un ciclo de vida definido: enviada, trabajando, requerimiento de entrada, completada, fallida, cancelada. La máquina de estados define qué significa cada estado, qué transiciones son válidas y qué datos intercambian los agentes en cada transición. La máquina de estados es lo que hace que un flujo de trabajo multiagente sea robusto ante fallas parciales.
