[ALLOY]: Los tiempos de espera por nodo son el ejemplo más claro. LangGraph agrega timeout= en add_node, con una TimeoutPolicy que puede establecer run_timeout, idle_timeout, o ambos. Un tiempo de espera inactivo se reinicia cuando se produce progreso y aborta cuando un nodo de transmisión deja de producir salida. Esa distinción importa para las herramientas de agentes. [NOVA]: DeltaChannel aborda la sobrecarga de puntos de control. En hilos de larga duración, canales como listas de mensajes crecen con el tiempo. Sin un mecanismo delta, cada punto de control puede re-serializar el valor acumulado completo. DeltaChannel almacena solo el delta incremental en cada paso y escribe una instantánea completa cada número configurado de pasos a través de snapshot_frequency. Eso cambia el modelo de costos. Las escrituras se vuelven más baratas para canales crecientes, mientras que las lecturas pueden necesitar reconstruir desde deltas hasta el próximo snapshot. La pregunta de ajuste es cuán frecuente hacer snapshots para que la latencia de reproducción permanezca limitada sin volver a la inflación de puntos de control de valor completo. ...

[ALLOY]: Esto es directamente relevante para sistemas de agentes porque historiales de mensajes, trazas de herramientas, listas de eventos y observaciones pueden crecer rápidamente.
