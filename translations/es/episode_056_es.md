[NOVA]: Los túneles MCP seguros convierten herramientas privadas en herramientas de agente accesibles sin abrir puertos de entrada.

[ALLOY]: Esta es una de las historias de infraestructura más importantes porque cambia cómo las herramientas privadas pueden participar en sistemas de agentes. Tanto OpenAI como Anthropic están avanzando en los patrones de túnel MCP. La forma básica es conectividad saliente: un servidor MCP local o interno se conecta a través de un túnel, y una superficie de agente puede alcanzarlo sin exponer un puerto de entrada a internet pública.

[NOVA]: La razón para importarte es simple. Muchas herramientas útiles son privadas. Viven en una laptop, dentro de una red corporativa, detrás de un firewall, o conectadas a un servicio local. Sin un túnel, un equipo no puede usarlas desde una superficie de agente hosted o tiene que hacer un agujero a través de la red. Un túnel saliente da una ruta de conectividad más limpia.

[ALLOY]: Pero conectividad no es permiso. Esa oración es todo el riesgo. Un túnel puede hacer que una herramienta sea alcanzable, pero no decide automáticamente quién debería usar la herramienta, qué métodos están permitidos, qué datos pueden salir, o qué acciones requieren aprobación. Si el servidor MCP expone acciones poderosas, el túnel solo acerca el problema.

[NOVA]: El ejercicio de "prueba ahora" es diseñar una herramienta MCP privada como si fuera a entrar en producción. Nombra la herramienta. Nombra la cuenta o proyecto que puede alcanzarla. Lista los métodos que el agente tiene permitido llamar. Decide cuáles llamadas son de solo lectura y cuáles son mutantes. Decide si las llamadas mutantes requieren aprobación humana. Registra cada llamada con suficiente detalle para auditarla después. Luego decide dónde vive el secreto local y cómo rota.

[ALLOY]: Eso no es burocracia. Es la diferencia entre "mi agente puede consultar un servicio interno de estado" y "mi agente hosted puede hacer cualquier cosa que exponga mi servidor MCP de laptop". El túnel debe ser estrecho. Si una herramienta solo necesita leer metadatos de issues, no le des un método que pueda reescribir el issue. Si solo necesita una base de datos de staging, no la apuntes a producción. Si solo necesita un resumen, no devuelvas registros crudos por defecto.

[NOVA]: Los mejores casos de uso son aburridos de buena manera: búsqueda interna, docs locales, estado de build, dashboards de solo lectura, verificaciones pequeñas de deployment, una herramienta privada de disponibilidad de calendario, o un inventario como fuente de verdad. Una vez que esos funcionen, muévete con cuidado hacia acciones de escritura. Una acción de escritura a través de un túnel privado debería sentirse como un pequeño release, no como una conveniencia casual.

[ALLOY]: La conclusión es práctica. Los túneles MCP seguros hacen más fácil conectar herramientas privadas. [NOVA]: Google Agent Executor hace que los agentes de larga duración sean retomables en lugar de ser de una sola vez.
